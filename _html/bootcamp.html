Bootcamp

The Bootcamp was a week in university in which we learned about the program processing

Bootcamp

Day 1 - 04/11/2016

Recapitulation

In the first “listen” segment we were introduced to the program and the the structure of the code. Furthermore, I learned how to test the code code and to visualize how it works using the “printnl” command (“Hello World”). The program would show e.g. a text, an integer, or the frame rate in the command box. The structure was very intuitive and easy to understand. Every code starts with a function, and ends with a semicolon (“ ; ”). The functions mostly consist out of the shape you want to create or the part of the shape you want to edit. Syntaxes that are used to create shapes would for example be; “ellipse(), rect(), triangle() etc.”. The brackets after the commands will contain the coordinates and other specifications of the shape. To further edit elements there are a number of attribute functions such as: “strokeWeight(), fill() or noStroke()” can be added in front of the shape function. Beside these attribute functions, we also learned how to use the RGB colour scale in our code. I learned about the writing style called camelCase which helps to read and distinct words from each other by capital letters at the beginning of a new word (eg. noStroke”) .

Furthermore I gained knowledge about the canvas itself. I learned how the canvas was made up, and how to position shapes on it. This knowledge helped when learning about flow and looping. Looping sketches with the “void draw() {}”  function, made the sketch more vivid and alive and also made it possible to create animations. Additionally I learned about variables and how to use them. Variables are basically containers capable of storing and containing data. This is very useful, when using a number more than once or when planing on randomising a number which will be used elsewhere in the code.

Assignment

The assignment was to create a sketch which had randomness implanted into it. My initial idea was to create a building that had lights whose brightness  depend on the sky's colour. The sky is supposed to sustain blue. In theory; the brighter the sky, the darker the lights in the building and the other way around. I was able to implant this function into my sketch by creating integers for each RGB colour of the sky and windows and + or - them by a randomised integer, that I called time. The time integer would pick a random number between zero and 100 and will either be added or subtracted from the RGB colour scheme (see image).  

After making the lights match the sky I started adding visuals like such as the hills the hills and suns (ellipses) to the sketch. To make it look more realistic I added transparency to the sun to create a shining effect. Furthermore, I wanted to add randomness to which of which lights would be switched on and which one is going to be switched off. In order to randomise the light for each light, I  created  an integer called “switchOnOff. The integer chooses a number between zero and two which I then us to multiply the RGB color scheme of the windows. This will either switch on the light or switch it of. Since I needed to create this line of code for every window I pasted it in front of every window (rect).

Day 2 - 07/11/2016

Recapitulation

On day two I gained insights on how to create interactive sketches. We learned about the concept of how frames correlate with the speed of the animation/sketch and how we can interact with it via controls of the keyboard and the mouse. In order to make the program more responsive to actions and values of variables, we learned about conditionals. Conditionals allow the program to make decisions based on statements which are expressed in an “if” or “else” function. The program evaluates the “test” that resolves to either be true or false and based on that executes or does not execute the statement.  The statement contains the lines of code that will be executed when the test resolved to be true. This is very useful when wanting to create an animation which has different outcomes.

The topic of how to add interactivity was a big part of the listening segment. I learned how the mouse coordinates can be used to for example move an object over the canvas. The interactivity of processing features all kind of input methods. We explored input methods from the keyboard, mouse buttons and mouse axis. Keyboard and mouse buttons can be used to display letters with “text()” or “key” commands or to give instructions of what to do if, while or when not pressed.

The order in which objects move over the canvas is called the flow of the program. Another topic that we discussed and learned about was keyboard data. This flow can be stopped by the “noLoop()” function. This function is useful to set an end to the animation, by the use of conditions.

Assignment

The assignment was to create a game. I like the idea of using the cursor to steer an object. I wanted to implement this functionality into the program which I did by using the mouse coordinates (mouseX and mouseY). I called the game“ The Roswell Crash” since the concept of the game is to steer the “Roswell UFO” through the incoming asteroids. The challenge is to navigate the object without crashing it too often.

I created a health bar which has an initial value of 1000 and illustrates the amount of health left.  Whenever the UFO collides with an asteroid the health bar shrinks depending on velocity of the obstacle and size. In addition to the subtraction of live the display turns black, which makes it harder to avoid the upcoming obstacles. For the collision detection I used the so called “dist()” function, that allows to measure the distance between two points. Whenever the two points get to close, the collision will be detected and the if statement turns true. When the if statement is true the health bar will shrink and the background will be set to: “(0)”. When the health bar is below zero the game will end with a noLoop() function and show an image, which was created on photoshop ( if (lives <=0) ). However the game has an objective, which is to prevent the Roswell crash from happening. For this I used the “text()” function to tell the user what the objective is and a kilometer counter in the top right corner. When the user makes it over 35,000 kilometers the game will end and show a different image. I added the photos by the Pimage function which allows to import images from the “data” folder.  To make the game looking aesthetically pleasing, I removed the cursor and added a background.

Day 3 - 08/11/2016

Recapitulation

The third day was about deepening the understanding of what we learned about the last days and about increasing efficiency in coding. The first listening part was also about understanding where to insert indents, and how to format the code in general. The  blocks of code, that shorten repetitive code into compact expressions are called iterate structures. We learned how to integrate them into our code and how to use it in different scenarios and with different objectives. The functions “while/for(test) {statement}” are used to loop or repeat lines of codes for a specific amount of time or loops. The difference between these types of loops is that a while loop always executes a code while the test reveals to be true and a for loop stops whenever a specific value  in the statement or of a integer is reached.

Furthermore I learned how to create functions, in order to shorten code and to create more complex sketches and animations. Creating functions makes the code unique and adapt to your needs. A self created function can for example contain specifics of what shape to display, and in what colour and with which stroke weight. Theses functions, can than be used multiple times and can be changed through variables (integers) inside the function. I also gained insights into how to calculate and when and how to return values. This can be helpful to return values of calculations into variables.

Assignment

The Assignment was to create an interactive sketch. I decided, to make my sketch interactive by the use of sounds and pictures, that respond to the mouse position. My idea was to create two buttons that as soon as you hover over them with the cursor change the background and sound. To implement sounds into the sketch I used the sound library by the “Processing Foundation”.  To activate the sounds I needed to type in the following command: “import processing.sound.*;” . Furthermore, I needed to give the sound file a name (SoundFile name;) and I needed to activate and give a path to the specific file by using the following command:  “name 0 new SoundFile (this, “name.mp3”);”.However to make it play or to stop playing the sound another command was needed another command. In order to make the sound stop playing I created two variables. One integer for when the ball is inside one of the circles and one for being in the other circle. Boths initial values were set to zero. By using an “else if”function I managed to only play the sound when the cursor is inside circle and stop the song as soon as the cursor is outside.

Furthermore I used a function to create the two circles and to set a mutual stroke weight and colour. Within the function I was able to set individual x and y coordinates and edit the size of the radius. The background image  is also responsive to the mouse position and correlates with the sound that is being played. Whenever no song is played the background turns to the black default background.

Day 4 - 09/11/2016

Recapitulation

On day four we learned about another method to store values. Arrays are basically sets of data that are stored under the same name and can contain numbers, characters, sentences, values, etc.. At first we learned how to define an array. Arrays are first declared by the command “int[] data = {data};” and can than be created with the keyword “new” in the set up part of the code. After the array is created the elements of the data set need to be assigned. Furthermore, we learned about the different ways  arrays can be declared, created and assigned. In this listening segment learned how the array works in a for loop based on examples inside the presentation. Furthermore I gained knowledge on how array elements are read by the computer and the software. The for loop function first initializes the array and tests the if the test turns true or false. If the test turns out to be true, the action inside the statement including the array data will be executed. In addition to that I learned how to use all data elements in one function. The command: “nameOfArray.length” will automatically include the whole length of the array into the code. This function is useful when creating a for loop that includes all data elements.

Assignment

The assignment for day 4 of the bootcamp was to create a game that has a main character or avatar and includes enemies or obstacles. In addition to that we were supposed to keep track of the score that was reached every time the game was played. I decided to create a car game called “Ghost Rider”. The scenario of the game is a car that drives on the wrong lane and needs to avoid approaching cars. The “ghost rider car” can be steered to the right and to the left side of the road. I used the “void keyPressed” command to integrate the arrow buttons into the game. Each time the player presses the left or right arrow key the car moves 60 pixels to equivalent direction. There are three different cars approaching the player, each car has it's on speed and randomised position on the street. Additionally I have added a function for the street lines. The street lines and cars spawn on the x coordinate 0 when they pass out of the canvas (x= 1500).  These visual create the feeling actually of travelling on a road. For this game I used the dist function again, to measure the distance between each car and the player's car. If the player gets to close to an approaching car the game ends with a “noLoop()” command and a red background is displayed. Whenever a car passes through the whole canvas, the score increases by one. The score is displayed in the console by the use of the “println()” function. Furthermore, the game features sounds. One sound is continues, throughout  the game and plays a sound o a car driving. The second sound is a crash sound, that is played when the player collides with a car.

Assignment

The fuel makes it more difficult to decide between avoiding a crash or collecting fuel (see image). The fuel bar ( rect() ) is responsive and changes every single frame and uses the value of the fuel tank for it's second x value.. The game ends when the fuel is empty and the player failed to collect enough petrol. This scenario and the scenario of a crash end the game, and were given a new background image and sound fitting to the scenario. Sounds were inserted through the library and included in the if functions to only play when appropriate

Final Reflection

When reflecting over the past days, it becomes clear that the bootcamp helped a lot to understand the basic process of creating software. Furthermore, it gave a many insights on the various building blocks used in processing.  The past days stimulated my creativity and gave me the opportunity to create animations and interactive sketches, that until this point I only could imagine. From day to day I gathered new insights that helped me to understand how I could have made the previous codes more efficient and complex. The very first code that I have written is very repetitive and does not contain iterate structures which underlined the process over the past days and the importance of complex and iterate structures. The past days were very interesting and contained a lot of information and inspiration. Furthermore, I think that processing will help a lot to illustrate ideas and concepts of interactions between software and users in the future. Overall the course changed my perception of programs and their complexity
